# Navis

## Product Purpose

Navis is a study in responsive dashboard navigation. The product exists to help developers choose and reuse polished mobile bottom navigation patterns that map cleanly from an existing desktop sidebar.

## Users

- Frontend developers building responsive dashboard web apps
- Product designers exploring mobile navigation behavior for web-based admin surfaces
- Teams that want reusable navigation patterns instead of rebuilding them per project

## Register

product

## Tone

Precise, calm, premium, and technical. The interface should feel designed by someone who understands product software, not by a marketing template.

## Design Principles

- Desktop sidebar and mobile bottom nav must feel like one information architecture
- The demo should showcase craft, not novelty for its own sake
- Royal blue is an accent, not the entire canvas
- Layout should feel editorial and deliberate, with strong rhythm and quiet confidence
- Motion should improve clarity and tactility, never draw attention to itself

## Anti-References

- Bright SaaS hero banners
- Generic card dashboards with repeated icon-heading-copy blocks
- Overuse of glass, gradients, and ornamental blur
- Crowded mobile nav bars with weak label legibility
